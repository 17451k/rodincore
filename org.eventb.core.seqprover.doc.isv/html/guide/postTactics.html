Adding a Post-tactic

Post-tactics are the tactics that run by after every manual proof steps in interactive mode. Only automatic tactics (See About tactics) can be declared as Post-tactic.

Post-tactics are added using the extension point org.eventb.core.postTactic. An example is as follows: 

   <extension
         point="org.eventb.core.postTactics">
      <tactic id="org.eventb.core.seqprover.trueGoalTac"/>
   </extension>

Each tactic element contains only attribute id which must be a valid tactic ID (including the project prefix) as declared using org.eventb.core.seqprover.autoTactics extension point.

Once declared as a Post-Tactic, the information about the tactic will appear in the preferences for the Post-Tactic and the users of the platform. However, this tactic will not be a default Post-Tactic and the users must change the preference to enable this tactic.
